# CameraPerf 新手上手指南

面向第一次接触本项目的开发者和使用者。目标：10 分钟跑起来，30 分钟理解核心概念。

## 项目是什么

CameraPerf 是基于 Google Perfetto 的 Android 性能分析平台。它在 Perfetto UI 的基础上增加了 AI 分析层：用自然语言提问，后端通过 Agent 编排 SQL 查询和预定义分析技能（Skill），给出结构化的性能诊断结论。

典型使用场景：
- 应用启动慢 → "分析启动性能"
- 滑动卡顿 → "分析滑动卡顿"
- ANR → "帮我看看这个 ANR"
- 渲染管线问题 → "RenderThread 为什么耗时长"

## 架构一览

```
┌─────────────────────────────────────────────────────────────┐
│  浏览器                                                       │
│  Perfetto UI (:10000) + AI Assistant 插件                     │
│    ├─ trace 加载 / 时间线 / SQL 查询                            │
│    ├─ AI 面板（Right / Bottom / 浮窗）                          │
│    └─ SSE 实时接收分析结果                                      │
└─────────────────────────┬───────────────────────────────────┘
                          │ HTTP / SSE
┌─────────────────────────▼───────────────────────────────────┐
│  后端 Express (:3000)                                         │
│    ├─ agentv3 运行时（Claude Agent SDK 编排）                   │
│    │    ├─ 20 个 MCP 工具（SQL、Skill、知识库、Schema 等）        │
│    │    ├─ 场景策略注入（12 个场景：滑动/启动/ANR/管线…）          │
│    │    └─ Verifier 验证 + 报告生成                             │
│    ├─ Skill 引擎（YAML 定义的确定性分析流水线）                   │
│    └─ trace_processor_shell（Perfetto 原生 SQL 引擎，RPC 池）   │
└─────────────────────────────────────────────────────────────┘
```

## 快速启动

### 方式一：Docker Hub 镜像（最快，不需要本地工具链）

```bash
git clone https://github.com/honor-99/CameraPerf.git
cd CameraPerf

# 配置 AI 模型（可选，见下方"模型配置"）
cp backend/.env.example .env
# 编辑 .env 填入 ANTHROPIC_API_KEY 或代理配置

# 拉取并启动
docker compose -f docker-compose.hub.yml pull
docker compose -f docker-compose.hub.yml up -d
```

### 方式二：本地源码运行（日常开发推荐）

前置要求：Node.js >= 24.0.0

```bash
git clone https://github.com/honor-99/CameraPerf.git
cd CameraPerf

# 启动（自动安装依赖 + 下载 trace_processor_shell）
./start.sh
```

### 方式三：本地源码 Docker 构建

```bash
# 用本地代码构建镜像并启动
docker compose up -d --build
```

### 启动后

| 服务 | 地址 |
|------|------|
| Perfetto UI | http://localhost:10000 |
| 后端 API | http://localhost:3000 |
| 健康检查 | http://localhost:3000/health |

## 模型配置

CameraPerf 基于 Claude Agent SDK，需要 Anthropic API 或兼容代理。

**场景 A：本地已有 Claude Code**

如果终端里 `claude` 命令已能正常工作，后端会自动使用 Claude Code 的本地认证，无需额外配置。

**场景 B：显式配置 API Key**

编辑 `backend/.env`（本地运行）或根目录 `.env`（Docker Hub 模式）：

```bash
# 直连 Anthropic
ANTHROPIC_API_KEY=sk-ant-xxx

# 或通过兼容代理（one-api / new-api / LiteLLM）
ANTHROPIC_BASE_URL=http://your-proxy:3000
ANTHROPIC_API_KEY=sk-proxy-xxx
CLAUDE_MODEL=your-model-name
```

**验证配置**

```bash
curl http://localhost:3000/health
# 看 aiEngine.configured 是否为 true
# 看 aiEngine.providerMode 是否正确
```

## 第一次分析

1. 打开 http://localhost:10000
2. 加载一个 `.pftrace` 文件（Android trace）
3. 打开 AI Assistant 面板（右侧或底部）
4. 选择分析模式：
   - ⚡ 快速：简单问题（包名、进程、基本数值），10 轮，成本低
   - 🔍 完整：复杂分析（启动、滑动、ANR），60 轮，结果深入
   - 🤖 智能：自动判断（默认推荐）
5. 输入问题，等待流式输出

**常用问题示例：**

```
分析滑动卡顿
分析启动性能
帮我看看这个 ANR
这个 trace 的应用包名和主要进程是什么？
CPU 调度有没有问题？
只看我选中的这段时间，为什么 UI thread 变慢？
```

## 项目结构（关键目录）

```
CameraPerf/
├── backend/                    # Express 后端（TypeScript）
│   ├── src/
│   │   ├── agentv3/           # 主运行时：Claude Agent SDK 编排
│   │   ├── services/          # 核心服务（trace processor、skill engine、报告）
│   │   ├── assistant/         # session 管理、stream projection
│   │   └── agent/             # 共享组件（检测器、上下文、核心接口）
│   ├── skills/                # YAML Skill 定义（原子/组合/深度/管线）
│   ├── strategies/            # 场景策略和 Prompt 模板（.md 文件）
│   ├── sql/                   # PerfettoSQL 自定义包
│   └── data/                  # SQL schema 索引、stdlib 符号表
├── frontend/                  # 预构建 Perfetto UI（不需要 submodule）
├── perfetto/                  # Perfetto submodule（仅 UI 开发时需要）
├── rust/flamegraph-analyzer/  # Rust 火焰图分析器（高性能路径）
├── scripts/                   # 启动、构建、更新脚本
├── test-traces/               # 回归测试用 trace 文件
└── docs/                      # 文档中心
```

## 核心概念

### Skill（技能）

YAML 定义的确定性分析流水线。Agent 通过 MCP 工具 `invoke_skill` 调用它们。

- 类型：atomic（单步）、composite（组合）、iterator（迭代）、parallel（并行）、conditional（条件）
- 结果分层：L1 概览 → L2 列表 → L3 诊断 → L4 深度
- 位置：`backend/skills/`

### Strategy（策略）

场景特定的系统 Prompt，用 Markdown 编写，YAML frontmatter 定义元数据。

- 12 个场景：scrolling、startup、anr、pipeline、interaction 等
- 位置：`backend/strategies/*.strategy.md`
- 不要在 TypeScript 中硬编码 Prompt 内容

### DataEnvelope

前后端统一数据 contract。Skill 输出的每个结果都包装成自描述的 DataEnvelope，前端根据 `display` 配置自动渲染表格、图表等。

### 分析模式路由

```
用户提问 → 场景分类（关键词匹配，<1ms）
         → 策略注入（对应场景的 .strategy.md）
         → 复杂度判断（keyword rules → hard rules → Haiku 分类器）
         → 选择 fast / full 路径
         → Agent 执行（MCP 工具 + Skill + SQL）
         → Verifier 验证
         → 结论 + 报告
```

## 开发工作流

### 后端改动

后端用 `tsx watch` 自动热重载，保存文件后刷新浏览器即可。

```bash
cd backend && npm run dev    # 或者用 ./start.sh 启动
```

### 前端插件改动

需要初始化 perfetto submodule 并使用开发脚本：

```bash
git submodule update --init --recursive
./scripts/start-dev.sh
```

改完后运行 `./scripts/update-frontend.sh` 更新预构建产物。

### 测试

```bash
# 必须：每次改动后跑回归
cd backend && npm run test:scene-trace-regression

# 类型检查
cd backend && npm run typecheck

# PR 前全量验证
npm run verify:pr

# Skill 验证
cd backend && npm run validate:skills

# 策略验证
cd backend && npm run validate:strategies
```

### CLI 工具

```bash
# 开发时
cd backend && npx tsx src/cli-user/bin.ts

# 构建后
cd backend && npm run build
node dist/cli-user/bin.js

# MCP server 模式（供 Claude Desktop 等集成）
node dist/bin/camerapref-mcp.js
```

## 推荐 trace 内容

CameraPerf 最适合 Android 12+ 包含 FrameTimeline 数据的 trace。

| 场景 | 必要 atrace category | 建议额外添加 |
|------|---------------------|-------------|
| 滑动 | `gfx`, `view`, `input`, `sched` | `binder_driver`, `freq`, `disk` |
| 启动 | `am`, `dalvik`, `wm`, `sched` | `binder_driver`, `freq`, `disk` |
| ANR | `am`, `wm`, `sched`, `binder_driver` | `dalvik`, `disk` |

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| "AI backend not connected" | 确认后端在 :3000 运行，检查 `/health` |
| 分析没有数据 | 检查 trace 是否包含必要的 atrace category |
| 端口冲突 | `pkill -f trace_processor_shell` 或换端口 |
| Node 版本不对 | 项目要求 Node >= 24.0.0 |
| Docker 构建慢 | 配置 Docker registry mirror；Perfetto artifact 下载可设 `TRACE_PROCESSOR_DOWNLOAD_BASE` |
| 调试 Agent 行为 | 查看 `backend/logs/sessions/*.jsonl` |
